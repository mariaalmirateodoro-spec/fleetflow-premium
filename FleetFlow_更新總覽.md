# FleetFlow Premium — 功能與修正總覽

更新日期:2026-07-08

## 新增功能

**訂單流程**
- 訂單草稿儲存(員工後台 + 客人自助表單皆可存草稿、之後續填)
- 客人email驗證機制,綁定自助訂單表單送出前的驗證流程
- 客人行程完成後可提交評分與留言(feedback)
- 司機「執行中」狀態自動判斷(依已核准訂單的接送時段自動顯示,不用手動切換)
- 返回客人搜尋自動帶入(員工建立新訂單時可搜尋舊客人資料,一鍵帶入姓名/電話/信箱等欄位)
- 供應商發票上傳功能

**報表與紀錄**
- 報表頁面(/reports)+ 報表快取(reports_cache)
- 系統活動紀錄頁面(Activity Log),記錄各項異動
- 訂單/報表 Excel 匯出(用 exceljs 產生正式格式的 Excel 檔)

**手機與 PWA**
- 後台管理畫面全面手機響應式改版:側邊欄改為可收合的抽屜選單、漢堡選單、通知面板、表格橫向滑動提示、狀態篩選列自動換行
- 訂單表單(員工後台 + 客人自助表單)修正手機上欄位重疊的問題
- 加入 PWA 安裝功能(可「加到主畫面」,像 App 一樣開啟)
- 網域首頁自動導頁:員工(已登入)進 Dashboard,客人(未登入)進車隊介紹頁 /fleet

**其他**
- 移除訂單表單中的「國籍」欄位(簡化客人填單流程)

## 修正的問題

**安全性(最重要)**
- 修復嚴重資料外洩漏洞:原本任何人不需登入就能讀取整張 bookings 表(所有客人姓名、電話、信箱、預算等)。已收回該權限,改為服務端限定查詢。
- 收緊 notifications 表的寫入權限,避免被任意寫入。

**正式站顯示異常**
- Bookings 頁面在正式站完全顯示不出訂單(0 筆、無錯誤訊息):根源是 feedback 表與 bookings 表的外鍵關聯在資料庫中缺失,導致整條查詢被 PostgREST 判定失敗、資料被靜默吞掉。已補上外鍵並修復查詢,評分資料也一併恢復顯示。

**效能**
- 修正每次切換頁面都重複打兩次「登入驗證」的問題(middleware 已驗證過一次,頁面又重打一次) —— 已合併成一次
- Dashboard 頁面原本一次要打 8 次資料庫查詢,其中多筆是重複抓同一批訂單資料 —— 精簡成 5 次,並讓「個人資料」與「儀表板資料」兩批查詢同時進行,不用排隊等待
- 加入 React cache() 避免同一次請求內重複查詢使用者資料
- 各頁面加上載入中骨架畫面(loading.tsx),避免切頁時畫面空白
- 字體改用 next/font 自行代管,避免額外的外部請求拖慢首次載入
- Approvals 頁面查詢改為平行處理,不用依序等待
- 通知系統改成掛在共用版型上,避免每次切頁都重新建立即時連線

**PWA / 手機**
- 修正客人把 /book 加到手機主畫面後,點開圖示卻被導去員工登入頁的問題:manifest 設定檔的啟動網址已改為智慧判斷首頁(比照網域首頁邏輯)
- 修正 manifest 設定檔本身被登入驗證攔截、導致格式錯誤/客人無法正常安裝的問題

## 目前已知、尚未處理的項目
- 訂單詳細頁的權限檢查與司機/供應商管理頁不完全一致,是否要收緊要看你的決定
- Dashboard 部分查詢未來訂單量變大後可能需要重新設計成資料庫端聚合計算,目前規模還不需要
